# 文档智能问答助手
一个基于本地大模型的学术文献智能问答系统，支持PDF和Word文档上传，通过RAG（检索增强生成）技术实现基于文档内容的精准问答，提供结构化的回答格式、流式响应输出与完整的聊天历史管理功能，所有数据处理均在本地完成，全面保障文档与数据隐私安全。

## 功能特性
### 核心功能
- 📄 **多格式文档支持**：支持PDF、DOCX、DOC格式文档批量上传与解析，双加载器兜底保障高兼容性
- 🧠 **本地模型推理**：基于Ollama运行本地大语言模型，无需调用外部API，数据全程不出本地
- 🔍 **精准RAG问答**：基于检索增强生成技术，仅从上传文档中提取信息回答，杜绝编造与臆测内容
- 📝 **结构化回答输出**：自动使用二级标题、加粗、列表等格式组织内容，专业术语标注、来源可追溯
- 💬 **多轮对话与历史管理**：完整保留聊天上下文，支持历史记录查看、快速跳转与一键清空
- ⚡ **流式实时响应**：边生成边返回回答内容，同步展示AI思考过程，无需等待完整结果
- 🔄 **全量会话重置**：支持一键清空所有上传文档、向量库与聊天记录，快速开启全新会话

### 技术特性
- 文档智能分块处理，优化长文档检索效率与上下文匹配精度
- 基于FAISS构建本地向量数据库，实现毫秒级相似性检索
- 支持多文档追加上传，自动合并构建统一向量检索库
- 响应式UI设计，适配不同屏幕尺寸，交互流畅友好
- 实时上传状态与解析结果提示，异常场景友好报错
- 内置平滑动画与流式渲染，提升使用体验
- 完整的API接口体系，支持二次开发与功能扩展

## 技术栈
### 后端
- Python 3.8+
- FastAPI：高性能异步API框架
- Uvicorn：ASGI web服务器
- LangChain：LLM应用开发核心框架
- langchain-community：文档加载器与生态组件
- langchain-ollama：Ollama模型集成适配
- FAISS：本地高效向量存储与检索引擎
- PyPDFLoader/UnstructuredPDFLoader：PDF文档双模式解析
- Docx2txtLoader/UnstructuredWordDocumentLoader：Word文档双模式解析
- RecursiveCharacterTextSplitter：智能文本分块工具

### 前端
- HTML5/CSS3
- JavaScript (ES6+)
- 原生CSS动画与交互实现
- SSE流式响应处理
- 无外部框架依赖，开箱即用

## 安装部署
### 前置条件
1. 安装 Python 3.8 及以上版本
2. 安装 Ollama 并下载所需模型
3. 建议配置：16GB及以上内存，8GB及以上显存（GPU）获得更优性能

#### Ollama 安装与模型准备
1. **Ollama 安装**
   - **Windows系统**：访问 [Ollama官方下载页](https://ollama.ai/download) 下载Windows安装包，按指引完成安装
   - **macOS系统**：终端执行以下命令安装，要求 macOS 14 Sonoma 及以上版本
     ```bash
     curl -fsSL https://ollama.com/install.sh | sh
     ```
2. **拉取所需模型**
   安装完成后，在终端/命令提示符中执行以下命令，拉取系统默认使用的模型：
   ```bash
   # 下载文本嵌入模型
   ollama pull qwen3-embedding:0.6b
   
   # 下载对话生成大模型
   ollama pull deepseek-r1:8b
   ```

### 项目部署步骤
1. **准备项目文件**
   将 `paper_read.py` 和 `chat.html` 两个文件放置在同一项目文件夹下（例如 `document-qa-assistant`）。

2. **创建并激活Python虚拟环境（推荐）**
   - **Windows系统**
     ```bash
     # 进入项目文件夹
     cd document-qa-assistant
     
     # 创建虚拟环境
     python -m venv venv
     
     # 激活虚拟环境
     venv\Scripts\activate
     ```
   - **macOS/Linux系统**
     ```bash
     # 进入项目文件夹
     cd document-qa-assistant
     
     # 创建虚拟环境
     python3 -m venv venv
     
     # 激活虚拟环境
     source venv/bin/activate
     ```

3. **安装Python依赖包**
   使用清华大学镜像源安装，速度更快更稳定：
   ```bash
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple fastapi uvicorn langchain langchain-community langchain-text-splitters langchain-ollama faiss-cpu pypdf docx2txt unstructured python-multipart pydantic
   ```

   也可创建 `requirements.txt` 文件批量安装，文件内容如下：
   ```
   fastapi>=0.104.1
   uvicorn>=0.24.0
   langchain>=0.1.0
   langchain-community>=0.0.19
   langchain-text-splitters>=0.0.1
   langchain-ollama>=0.1.0
   faiss-cpu>=1.7.4
   pypdf>=3.17.0
   docx2txt>=0.8
   unstructured>=0.10.30
   python-multipart>=0.0.6
   pydantic>=2.0.0
   ```
   执行安装命令：
   ```bash
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
   ```

4. **启动应用服务**
   ```bash
   python paper_read.py
   ```

5. **访问系统**
   启动成功后，终端会显示访问地址提示，打开浏览器（推荐Chrome、Edge、Firefox）访问：
   ```
   http://localhost:8000
   ```

## 核心配置说明
系统所有核心参数均可在 `paper_read.py` 文件顶部的「全局配置参数」区块修改，配置项明细如下：

| 配置项 | 默认值 | 功能说明 |
|--------|--------|----------|
| EMBEDDING_MODEL | qwen3-embedding:0.6b | 用于文档/查询向量化的嵌入模型名称 |
| LLM_MODEL | deepseek-r1:8b | 用于生成回答的大语言模型名称 |
| LLM_TEMPERATURE | 0.1 | 模型生成温度，值越低回答越严谨、随机性越小 |
| LLM_NUM_CTX | 4096 | 模型上下文窗口大小，控制可处理的文本长度 |
| LLM_NUM_GPU | 999 | 调用的GPU数量，默认全量使用GPU资源 |
| CHUNK_SIZE | 1000 | 文档分块的单块最大字符数 |
| CHUNK_OVERLAP | 100 | 相邻文档块的重叠字符数，避免上下文断裂 |
| SEARCH_K | 5 | 单次检索返回的相关文档块数量 |
| SERVER_HOST | 0.0.0.0 | 服务监听地址，默认支持所有本地/局域网访问 |
| SERVER_PORT | 8000 | 服务监听端口号 |

## 快速使用指南
### 基础使用流程
1. **上传文档**
   - 点击界面「上传文档」按钮，选择1个或多个PDF/DOCX/DOC格式文件
   - 等待文档解析完成，界面会显示上传状态与成功解析的文件列表
   - 若提示「当前不支持该文件类型，请尝试其他文件」，请确认文件格式为支持的PDF/Word格式

2. **发起提问**
   - 在底部输入框中输入关于上传文档的问题，支持Shift+Enter换行
   - 按下Enter键或点击发送按钮提交问题
   - 界面会实时展示AI思考过程与流式生成的回答内容

3. **查看回答结果**
   - 回答自动按规范结构化展示，关键概念、结论自动加粗标注
   - 多要点内容自动使用列表格式组织，提升可读性
   - 回答末尾统一标注信息来源文档名称

### 高级功能使用
#### 聊天历史管理
- 右侧边栏展示所有历史提问记录
- 点击任意历史记录，可快速跳转到对应的对话位置
- 点击「清空历史」按钮，可一键删除所有聊天记录，保留已上传的文档

#### 会话管理
- 点击「重置」按钮：一键清空所有上传文档、向量数据库、聊天记录与临时文件，完全重置系统
- 点击「清空历史」按钮：仅删除聊天对话记录，已上传的文档与向量库保留，可继续基于文档提问

## 系统核心工作原理
### 1. 文档处理全链路
1. 上传的文档保存到本地临时目录
2. 根据文件后缀自动选择对应加载器，双加载器兜底保障解析成功率
3. 为每个文档片段添加来源文件名元数据，实现溯源能力
4. 使用递归字符分切器，按配置的chunk_size与chunk_overlap将长文档切分为标准化文本块
5. 调用嵌入模型，为每个文本块生成向量表示
6. 向量数据存入FAISS本地向量数据库，构建检索索引

### 2. 问答处理全链路
1. 接收用户提问，调用嵌入模型生成问题向量
2. 从FAISS向量库中检索与问题最相关的TOP-K个文档块
3. 将检索到的文档上下文、聊天历史、系统提示词与用户问题整合，送入LLM
4. 流式接收LLM生成的内容，自动解析思考标签与正式回答内容
5. 实时将内容推送到前端渲染，完成后将完整对话存入聊天历史

### 核心组件说明
- **PaperRAG核心类**：实现RAG全流程逻辑，包含文档加载、向量库构建、流式问答、会话管理等核心方法
- **FastAPI服务层**：提供标准化HTTP接口，包含文档上传、流式问答、历史清空、会话重置等核心端点
- **前端交互层**：实现文件上传、流式消息渲染、历史记录管理、用户交互等功能，无需额外依赖

## 自定义配置与扩展
### 模型配置修改
直接修改全局配置参数中的 `EMBEDDING_MODEL` 与 `LLM_MODEL`，可替换为任意Ollama支持的模型，示例：
```python
# 替换为Llama3系列模型
LLM_MODEL = "llama3:8b"
EMBEDDING_MODEL = "nomic-embed-text:latest"
```

### 文档分块配置优化
根据文档类型与长度，可调整分块参数，适配不同场景：
```python
# 长文档/书籍优化配置
CHUNK_SIZE = 1500
CHUNK_OVERLAP = 200

# 短文档/论文摘要优化配置
CHUNK_SIZE = 500
CHUNK_OVERLAP = 50
```

### 回答规则与格式定制
修改 `SYSTEM_PROMPT` 系统提示词，可自定义AI助手的角色定位、回答格式、输出规范、语言风格等。

## 常见故障排除
### 1. 模型相关问题
- **模型拉取失败**：检查网络连接，确认Ollama服务正常运行，可尝试更换镜像源或手动下载模型
- **模型加载失败**：执行 `ollama list` 确认模型已正确下载，核对配置文件中的模型名称是否正确，重启Ollama服务后重试
- **GPU未被调用**：确认显卡驱动与CUDA环境正常，检查Ollama是否已启用GPU支持，调整 `LLM_NUM_GPU` 参数

### 2. 文档解析相关问题
- **提示不支持的文件类型**：确认文件后缀为 `.pdf`/`.docx`/`.doc`，避免使用加密、损坏或特殊格式的文档
- **文档解析失败**：对于加密PDF请先解密后上传，复杂格式Word文档可另存为标准格式后重试，检查文件是否被其他程序占用
- **文档分块异常**：调整 `CHUNK_SIZE` 与 `CHUNK_OVERLAP` 参数，避免文档内容过短或格式混乱

### 3. 服务与依赖相关问题
- **依赖安装失败**：确保使用清华大学镜像源，检查Python版本是否符合要求，逐个安装依赖定位失败的包，更新pip后重试
- **端口被占用**：修改配置中的 `SERVER_PORT` 参数，更换为未被占用的端口号
- **服务启动失败**：检查虚拟环境是否正确激活，所有依赖是否安装完成，核对Python版本是否为3.8及以上

### 4. 问答相关问题
- **回答与文档无关**：确认问题与上传文档内容相关，检查文档是否解析成功，调整 `SEARCH_K` 参数增加检索上下文，降低模型温度
- **回答内容编造**：严格遵循系统提示词规则，确保上下文完整传入，降低 `LLM_TEMPERATURE` 至0.1以下
- **流式响应异常**：检查浏览器是否支持SSE，关闭代理/VPN后重试，确认服务端与客户端网络连通正常

### 5. 性能相关问题
- **响应速度慢**：使用参数量更小的模型，降低 `LLM_NUM_CTX` 与 `SEARCH_K` 参数，关闭其他占用内存/显存的程序
- **内存不足**：减小文档分块大小，降低上下文窗口长度，分批上传大文档，增加系统虚拟内存
- **检索速度慢**：优化文档分块数量，使用FAISS GPU版本，减少单次上传的文档总量

## 注意事项
1. **数据安全**：所有文档与对话数据均存储在本地临时目录，重置会话会自动清理临时文件，无数据泄露风险
2. **硬件要求**：建议使用16GB及以上内存、8GB及以上显存的设备运行，大模型推理对硬件配置有一定要求
3. **文档处理**：超大文档建议拆分后分批上传，避免单次处理占用过多系统资源；建议使用英文文件名，避免特殊字符导致解析异常
4. **网络要求**：仅首次下载模型与依赖包需要网络连接，正常使用过程无需联网，完全离线运行
5. **系统兼容性**：Windows、macOS、Linux系统均支持，macOS需升级至14 Sonoma及以上版本
6. **使用规范**：提问越具体，回答越精准，避免过于宽泛的问题；系统仅基于上传文档内容回答，无相关信息时会明确说明，不会编造内容
7. **数据备份**：重要文档请提前备份，会话重置后所有上传文档与聊天记录将被清空且无法恢复
8. **长期使用**：长时间使用后，建议定期重置会话清理临时文件，释放磁盘空间与系统资源
